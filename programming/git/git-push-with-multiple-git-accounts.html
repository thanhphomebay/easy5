Git Push - Dealing With Multiple Github Accounts In The Same Machine

Like me you also have multiple github accounts. It is a hassle to deal with git push. The key to solve this issue is the ssh configuration file. When you do a git push (using ssh instead of https protocol), the ssh configuration file will replace the host firstacc with github.com and it also looks for the private key id_rsa.firstacc to authenticate with the id_rsa.pub on github account. Below are steps to aleviate the issue.

  1. Generate ssh key pair
    For each github account, generate an ssh key pair: ssh-keygen -t rsa -C "your github id"
  2. Copy the public key to your github account, rename the private key ~/.ssh/id_rsa to ~/.ssh/id_rsa.firstacc
  3. Create SSH configurationi
      cat ~/.ssh/config
      # 1st account
      Host firstacc
      HostName github.comp
      User git
      IdentityFile ~/.ssh/id_rsa.firstacc
      # 2nd account
      Host secondacc
      HostName github.com
      User git
      IdentityFile ~/.ssh/id_rsa.secondacc
  4. git push
    1. cd "your 1st project"
    2. git init
    3. git config user.email "your email"
    4. git config user.name "your name"
    5. git remote add origin git@firstacc:your1stProjectGithubId/yourfirstproject.git
    6. git push -u origin master
    7. git add .
    8. git commit . -m "init"
  5. Repeat step 1 to 3 for the 2nd, 3rd and so forth github accounts you may have
  6. Done :)
